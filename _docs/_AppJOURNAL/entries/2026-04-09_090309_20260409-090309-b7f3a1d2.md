- Title: Reusable inference loop cartridge and loop slot

## Summary

Added a reusable inference loop architecture so UsefulHELPER no longer hardwires its local inference path directly inside the manager. The worker now owns a manager-level loop slot backed by `InferenceLoopService`, ships a default `ollama.single_turn` cartridge, exposes `inference.describe_loops`, and routes both `ollama.chat_json` and `ollama.chat_text` through the same reusable loop seam.

## Notes

- The loop seam is now explicit and reusable instead of being buried in one Ollama route.
- `ollama.list_models` remains a direct bounded inventory helper while request/response loops go through the slot.
- The current default cartridge is `ollama.single_turn`, which keeps the lane simple while making future swaps or additions straightforward.
